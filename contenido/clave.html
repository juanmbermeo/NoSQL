Redis

Concepto

Redis es un almacén de estructura de datos de valores de clave en memoria rápido y de código abierto. Redis incorpora un conjunto de estructuras de datos en memoria versátiles que le permiten crear con facilidad diversas aplicaciones personalizadas. Entre los casos de uso principales de Redis se encuentran el almacenamiento en caché, la administración de sesiones, pub/sub y las clasificaciones. Es el almacén de valores de clave más popular en la actualidad. Tiene licencia BSD, está escrito en código C optimizado y admite numerosos lenguajes de desarrollo. Redis es el acrónico de REmote DIctionary Server (servidor de diccionario remoto).

Gracias a su velocidad y facilidad de uso, Redis es una opción popular para aplicaciones web, móviles, de juegos, de tecnología publicitaria y de IoT que requieren el mejor desempeño de su clase. AWS proporciona compatibilidad con Redis mediante un servicio de base de datos totalmente gestionado y optimizado llamado Amazon ElastiCache para Redis, y además permite a los clientes ejecutar Redis en AWS EC2 administrado por ellos mismos.

Características

Desempeño rápido

Todos los datos de Redis se encuentran en la memoria principal del servidor, a diferencia de la mayoría de sistemas de administración de bases de datos, que almacenan los datos en el disco o en SSD. Al eliminar la necesidad de acceder a discos, las bases de datos en memoria como Redis evitan los retrasos y pueden acceder a los datos con algoritmos más sencillos que utilizan menos instrucciones de la CPU. Las operaciones típicas tardan menos de un milisegundo en ejecutarse.

Estructura de datos en memoria

Redis permite a los usuarios almacenar claves que se corresponden con diversos tipos de datos. El tipo de datos fundamental es una cadena, que puede componerse de texto o datos binarios y tener un tamaño de hasta 512 MB. Redis también admite listas de cadenas en el orden en el que se han agregado, conjuntos de cadenas sin ordenar, conjuntos clasificados ordenados por puntuación, hashes que almacenan una lista de campos y valores, e HyperLogLogs que cuentan los elementos únicos de un conjunto de datos. Con Redis, se puede almacenar en la memoria prácticamente cualquier tipo de datos.

Replicación y persistencia

Redis utiliza una arquitectura maestro-esclavo y admite la replicación asíncrona mediante la que los datos se replican en numerosos servidores esclavos. De este modo, se logra una mejora en el desempeño de lectura (ya que las lecturas se pueden repartir entre servidores) y de recuperación cuando el servidor principal sufre un fallo.
Para proporcionar durabilidad, Redis admite las snapshots de un momento determinado (copiando el conjunto de datos de Redis en un disco) y la creación de un archivo de solo anexos (AOF) para almacenar cada uno de los cambios a los datos en un disco a medida que se producen. Ambos métodos facilitan la recuperación rápida de los datos de Redis si se produce un fallo.

Compatibilidad con casi cualquier lenguaje

Los que quieran desarrollar en Redis, pueden hacerlo con multitud de lenguajes de programación, como por ejemplo Java, Python, PHP, C, C++, C#, JavaScript, Node.js, Ruby, R, Go y muchos más.

Versatilidad y facilidad de uso

Redis incorpora varias herramientas que facilitan y aceleran el desarrollo y las operaciones, incluidas Pub/Sub, para publicar mensajes en canales, que se entregan a suscriptores, lo que es ideal para sistemas de chat y mensajería; las claves TTL, que indican un tiempo de vida determinado, tras el que se eliminan a sí mismas, lo que resulta útil para evitar llenar las bases de datos de datos no necesarios; los contadores atómicos, para garantizar que las condiciones de la carrera no creen resultados incoherentes; y Lua, un lenguaje de scripts ligero pero potente. Además, es muy fácil de utilizar, gracias a las diferentes herramientas que aceleran el desarrollo, como:
Pub/Sub, fantástico para sistemas de mensajería instantánea y que permite publicar mensajes en canales para entregar a los suscriptores.
Claves TTL, que se destruyen a sí mismas con el tiempo que le indiquemos y que son ideales para evitar datos innecesarios en nuestra base.
Contadores atómicos, que nos aseguran que no se creen resultados incoherentes.
Lua, un lenguaje de scripts liviano pero poderoso.

Funcionamiento

Redis almacena todos los datos en la memoria RAM, lo que le permite ofrecer un rendimiento muy rápido. Los datos se almacenan en estructuras de datos simples y eficientes, como cadenas, listas, conjuntos y diccionarios. Redis también admite transacciones atómicas, lo que significa que se pueden realizar varias operaciones en los datos de forma segura y consistente.
Redis es una base de datos distribuida, lo que significa que se puede ejecutar en varios servidores. Esto permite escalar Redis horizontalmente para manejar grandes cargas de trabajo. Redis también admite la replicación, lo que significa que los datos se pueden replicar en varios servidores para garantizar la alta disponibilidad.

Almacenamiento de datos:
En memoria

Redis almacena todos los datos en la memoria RAM, lo que proporciona un acceso extremadamente rápido a la información. Esto la diferencia de las bases de datos relacionales tradicionales que guardan datos en el disco duro, lo que puede generar latencias más altas.

Estructura de datos eficientes

Redis utiliza estructuras de datos simples y eficientes como cadenas, listas, conjuntos y diccionarios para organizar la información. Estas estructuras optimizan el acceso y la manipulación de datos, mejorando el rendimiento general.

Arquitectura distribuida:
Escalabilidad horizontal

Redis se puede distribuir en múltiples servidores, permitiendo aumentar su capacidad de procesamiento y almacenamiento de datos de forma horizontal. Esto la hace ideal para aplicaciones que manejan grandes volúmenes de información o picos repentinos de tráfico.

Alta disponibilidad

Redis admite la replicación de datos entre servidores, asegurando que la información permanezca accesible incluso en caso de fallos en un nodo. Esta característica garantiza la continuidad del servicio y la protección contra la pérdida de datos.

Modelo de datos clave-valor:
Asociación simple

Redis asocia cada dato con una clave única, permitiendo un acceso rápido y directo a la información. La simplicidad del modelo clave-valor facilita la recuperación y manipulación de datos, optimizando el rendimiento de las consultas.

Flexibilidad de datos

Las claves en Redis pueden contener cadenas, listas, conjuntos y diccionarios, ofreciendo una gran flexibilidad para almacenar y organizar información diversa. Esta versatilidad permite adaptar la base de datos a las necesidades específicas de cada aplicación.

Transacciones atómicas:
Operaciones consistentes

Redis soporta transacciones atómicas, garantizando que un conjunto de operaciones se ejecute de forma completa o no se ejecute en absoluto. Esto previene la inconsistencia de datos y asegura la integridad de las transacciones.

Aislamiento de Operaciones

Las transacciones atómicas aíslan las operaciones entre sí, evitando conflictos y garantizando la ejecución segura y consistente, incluso en entornos con múltiples usuarios o procesos accediendo simultáneamente a la base de datos.

Persistencia de datos:
Opcional

Redis ofrece la opción de persistir los datos en el disco duro mediante la función AOF (Append-Only File) o RDB (Snapshotting). Esto permite recuperar la información en caso de un reinicio inesperado del servidor.

Volatilidad

Redis también puede configurarse para funcionar en modo volátil, donde los datos se almacenan únicamente en memoria y se pierden al reiniciar el servidor. Esta opción es útil para aplicaciones que no requieren persistencia de datos a largo plazo.

Desventajas

Limitaciones de almacenamiento:
Dependencia de la memoria RAM

Al estar basado en memoria, Redis tiene un límite de capacidad definido por la cantidad de RAM disponible en el servidor. Si la cantidad de datos crece significativamente, puede ser necesario aumentar la memoria RAM o implementar técnicas de compresión de datos.

Costos de infraestructura

El uso de grandes cantidades de memoria RAM puede implicar costos de infraestructura más elevados, especialmente en entornos de producción con altos requerimientos de rendimiento.

Naturaleza volátil:
Pérdida de datos

Por defecto, Redis funciona en modo volátil, lo que significa que los datos se pierden al reiniciar el servidor. Si se requiere persistencia de datos a largo plazo, es necesario habilitar la funcionalidad de persistencia mediante AOF o RDB, lo que puede afectar el rendimiento.

Riesgo de pérdida de información

En caso de fallos en la alimentación o errores de software, los datos en memoria podrían perderse si no se ha configurado la persistencia adecuada.

Soporte para consultas complejas:
Modelo simple

Redis está diseñado para operaciones simples de lectura y escritura, y no es ideal para realizar consultas complejas que involucran múltiples conjuntos de datos o uniones. Para este tipo de tareas, se recomienda utilizar bases de datos relacionales o soluciones de análisis de datos especializadas.

Limitaciones de filtrado

Las capacidades de filtrado y búsqueda de Redis son básicas, lo que puede dificultar la recuperación de información específica de grandes conjuntos de datos.

La gestión de clusters de Redis distribuidos puede ser más compleja que la de bases de datos monolíticas (Una arquitectura monolítica es un modelo de desarrollo de software tradicional que utiliza un código base para realizar varias funciones empresariales. Todos los componentes de software de un sistema monolítico son interdependientes debido a los mecanismos de intercambio de datos dentro del sistema), requiriendo conocimientos técnicos y herramientas especializadas para garantizar el correcto funcionamiento y la alta disponibilidad.
En comparación con bases de datos relacionales como MySQL o PostgreSQL, Redis tiene un ecosistema de herramientas y bibliotecas de menor tamaño, lo que puede dificultar encontrar soluciones a problemas específicos o integrar la base de datos con otras aplicaciones.
